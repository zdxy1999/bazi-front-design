# 姻缘匹配页面测试指南

## 问题排查

如果match页面显示不正常，请按以下步骤排查：

### 1. 清除浏览器缓存
- 按 `Ctrl + Shift + Delete` (Windows) 或 `Cmd + Shift + Delete` (Mac)
- 清除"缓存的图片和文件"
- 或者使用**无痕/隐私模式**打开页面

### 2. 正确的访问路径
**不要直接访问 match.html**，应该从首页进入：

1. 打开首页：`src/pages/home/home.html`
2. 找到一个有推荐结果的档案（profile_001 坎卦用户有推荐）
3. 点击圆形菜单 → "姻缘匹配"按钮（红色心形图标）
4. 页面会跳转到：`match.html?profileId=profile_001`

### 3. 查看Console日志
打开浏览器开发者工具（F12），查看Console标签：

**应该看到**：
```
[Match] 初始化参数: {profileId: "profile_001", status: undefined}
[Match] 正在加载档案: profile_001
[Match] 档案加载成功: {name: "用户_坎卦", matchStatus: "HAS_RESULTS", matchResultsCount: 3}
[Match] 开始渲染推荐结果，数量: 3
[Match] 渲染推荐 1: 推荐对象1
[Match] 渲染推荐 2: 推荐对象2
[Match] 渲染推荐 3: 推荐对象3
[Match] 推荐卡片渲染完成
[Match] 渲染第一个推荐的匹配理由
[Match] swiper已初始化，添加slideChange监听
```

**如果看到错误**：
- `缺少档案ID` → 说明访问方式不对，需要从首页进入
- `找不到档案` → 说明profileId不正确
- `没有推荐结果` → 说明该档案还没有匹配数据

### 4. 确认Mock数据
检查 `mock-data/bazi-profiles.json` 文件：
- profile_001 应该有 `matchStatus: "HAS_RESULTS"`
- profile_001 应该有 `matchResults` 数组，包含3个推荐对象
- 每个推荐对象应该有完整的 `bazi`、`birthDate`、`minguaName`、`reason` 字段

### 5. 测试步骤
1. 确保已经运行了本地服务器（不能用file://协议）
   ```bash
   # 在design-20260406目录下
   python3 -m http.server 8000
   # 或
   npx serve
   ```

2. 访问首页：
   ```
   http://localhost:8000/src/pages/home/home.html
   ```

3. 点击圆形菜单的红色心形图标（姻缘匹配）

4. 应该会跳转到：
   ```
   http://localhost:8000/src/pages/match/match.html?profileId=profile_001
   ```

### 6. 预期结果
- 看到3个推荐卡片可以左右滑动
- 每个卡片显示：
  - 命卦图标
  - 推荐对象名称
  - 出生日期
  - 八字信息（4个干支）
  - 命卦标签
- 下方显示匹配理由（Markdown格式渲染）
- 点击"返回"和"重新计算"按钮都正常

## 常见错误

### 错误1：`Uncaught SyntaxError: Unexpected token 'export'`
**原因**：浏览器缓存了旧版本的processing.js
**解决**：清除缓存，我已经修复了这个问题（注释掉了export）

### 错误2：`marked is not defined`
**原因**：Markdown依赖没有加载
**解决**：我已经添加了CDN链接，确保网络可以访问：
- `https://cdn.jsdelivr.net/npm/marked/marked.min.js`
- `https://cdnjs.cloudflare.com/ajax/libs/highlight.js/11.9.0/highlight.min.js`

### 错误3：`Cannot read properties of undefined (reading 'yearPillar')`
**原因**：推荐对象的bazi数据不存在
**解决**：我已经添加了安全检查，会显示"八字信息加载中..."

### 错误4：swiper slides数量为0
**原因**：matchResults数组为空或不存在
**解决**：确认使用的是profile_001，它有3个推荐结果

  id: "profile_001",
  name: "测试用户",
  matchStatus: "HAS_RESULTS",
  matchResults: [

      birthDate: "1992-03-15T10:30:00",
      bazi: {
        yearPillar: { stem: "壬", branch: "申" },
        monthPillar: { stem: "甲", branch: "寅" },
        dayPillar: { stem: "乙", branch: "卯" },
        hourPillar: { stem: "丙", branch: "辰" }
      },
